# Brand Assets

## Purpose

This document defines the official brand asset categories, asset usage rules, and asset governance requirements for Relvio.

It ensures that approved Relvio brand assets are used consistently across:

- The Relvio product
- Websites
- Documentation
- Presentations
- Marketing materials
- Social media
- Product distribution platforms

This document does not redefine the Relvio brand identity or redesign the Relvio logo.

Brand identity decisions are governed by:

- `19_Brand_Identity.md`

Logo strategy and logo usage decisions are governed by:

- `20_Logo_Strategy.md`

Flutter production asset organization is governed by:

- `Asset_Structure.md`

If a conflict exists, the approved source document for that subject takes precedence.

# Official Brand Name

The official public product name is:

**Relvio**

Always write the product name exactly as:

```text
Relvio



Do not use public-facing variations such as:

relvio
RELVIO
RelVio
Relvio OS
Atlas

Atlas was an internal early codename and must not be presented as the public product name.

Primary Brand Message

The approved primary brand message is:

Build stronger relationships.

This is the primary brand-facing message for Relvio.

The following phrase may be used as a product category descriptor where context requires it:

People Operating System

The category descriptor is not a replacement for the primary brand message.

Do not introduce new taglines or primary brand messages without explicit brand approval.

Approved Logo Identity

The approved Relvio logo is the custom Relvio R inside a connected orbital system.

The logo identity includes:

A custom Relvio R
Connected orbital relationships
Relationship nodes
Intentional orbital breaks

These elements are part of the approved visual identity.

The logo must not be:

Redrawn
Reinterpreted
Reconstructed from screenshots
Recreated with Flutter drawing code
Recreated with CSS
Recreated with SVG paths generated by an AI coding assistant
Replaced with a generic letter R
Replaced with a generic network or connection symbol

Approved logo source assets must be used directly.

If an approved logo asset is unavailable, the missing asset must be reported.

The asset must not be invented.

Approved Logo Assets

The official brand asset library may contain approved exports for:

Primary logo
Logo mark
Wordmark
Horizontal logo arrangement
Monochrome logo
White logo
App icon

A logo variation is official only when an approved source asset exists.

The existence of a variation in this document does not authorize a developer or AI coding assistant to generate it.

Missing variants must be reported to Brand & Design.

Logo Source of Truth

The approved master logo source is the source of truth for all logo exports.

Production exports must originate from the approved master asset.

Do not use:

Screenshots
UI mockup crops
Compressed social media images
AI-generated approximations
Traced screenshots
Manually reconstructed logo geometry

as production logo sources.

App Icon

The Relvio app icon must use the approved Relvio logo identity.

The app icon must:

Remain recognizable without supporting text
Preserve the approved logo geometry
Remain visually clear at platform icon sizes
Follow Android and iOS distribution requirements
Use approved brand colors
Be exported from an approved source asset

The app icon must not be recreated from a screenshot or UI mockup.

Platform-specific icon exports may be generated from the approved master app icon source.

Favicon

Website favicon assets should be generated from the approved Relvio logo mark.

Required favicon sizes may include:

16×16
32×32
48×48

Additional platform-specific favicon or web app icon sizes may be exported when required by the target platform.

All favicon assets must originate from the approved logo source.

Logo Clear Space

The Relvio logo must retain sufficient clear space to remain visually distinct from surrounding content.

Clear-space measurements must follow the approved logo source or documented logo specification.

If an exact clear-space measurement has not been formally defined in the approved logo source, developers and AI coding assistants must not invent a geometric spacing formula.

Use the approved exported asset with appropriate surrounding layout spacing.

Minimum Logo Size

Logo assets must remain visually legible at the size in which they are used.

Platform requirements and visual clarity determine the final rendered size.

Do not apply arbitrary universal minimum sizes unless those measurements have been formally approved in the logo specification.

For product UI, use the dimensions defined by the approved Relvio UI design.

For app distribution, websites, presentations, and print materials, validate logo clarity in the target medium.

Feature-specific versions

Typography

The approved Relvio typeface is:

Inter

Inter should be used across supported Relvio brand and product surfaces where technically appropriate.

Platform or system fallback fonts may be configured for resilience.

Do not introduce additional primary brand typefaces without explicit approval.

Typography implementation must follow the approved Relvio UI and design token system.

Iconography

Product iconography must remain:

Minimal
Clear
Consistent
Appropriate for mobile interfaces

Icons must match the approved Relvio UI.

Do not introduce a new icon style or icon library merely because this document references iconography.

The implementation team must use the icon source or library selected during Flutter implementation based on the approved UI.

AI coding assistants must not replace approved icon intent with unrelated decorative icons.

Illustration Assets

Illustrations are not required by default.

If an approved Relvio illustration exists, the approved asset must be used directly.

Do not generate or introduce illustrations merely to fill:

Empty states
Error states
Loading states
Onboarding screens

The approved Relvio UI determines whether an illustration is required.

Missing required illustrations must be reported to Design.

They must not be invented by an AI coding assistant.

Photography

When photography is used in Relvio brand or marketing materials, it should prioritize:

Real people
Genuine human interaction
Diverse communities
Authentic environments
Natural visual treatment

Avoid imagery that feels:

Artificial
Overly staged
Generic
Unrelated to people and relationships

Photography guidance applies primarily to brand and marketing surfaces.

It does not authorize the addition of photography to the approved Relvio mobile UI.

Animation Assets

Lottie and Rive assets are not required by default.

Simple animations and micro-interactions should be implemented directly in Flutter where appropriate.

Animation asset libraries may only be introduced when:

The approved UI requires a complex animation
Direct Flutter implementation is unsuitable
The dependency and asset are reviewed
The implementation does not conflict with the approved architecture

AI coding assistants must not add Lottie, Rive, or decorative animation packages automatically.

Asset_Structure.md

Flutter Asset Usage

Approved production brand assets used by Flutter must be placed in the correct production asset directory defined by Asset_Structure.md.

Flutter code must reference centralized asset constants.

Use:

AppAssets

Do not scatter raw asset path strings throughout widgets or features.

Before using a brand asset, implementation must verify that:

The asset exists.
The asset is approved.
The asset is stored in the correct production directory.
The asset is registered correctly for Flutter.
The asset is referenced through the approved asset constant structure.

If an expected approved asset is missing, report the missing asset.

Do not generate a substitute.

AI Coding Assistant Rules

AI coding assistants, including Claude and similar implementation tools, must treat approved brand assets as immutable source materials.

AI coding assistants must not:

Redesign the Relvio logo
Generate a replacement logo
Trace the logo from screenshots
Draw the logo using Flutter CustomPainter
Generate logo SVG paths
Invent missing brand assets
Invent brand colors
Introduce new typography
Add decorative illustrations without approval
Crop production assets from UI screenshots
Add animation asset libraries by default

When an approved asset is missing, the implementation assistant must:

Stop the affected asset-specific implementation.
Identify the missing asset.
Report the expected asset location or requirement.
Continue only with unrelated implementation work that does not require inventing the missing asset.
Asset Version Control

Approved brand asset updates should record:

Version number
Date
Description
Owner

Superseded assets should be archived.

Production code must not continue referencing superseded assets after an approved migration is completed.

Do not silently overwrite approved master assets with reconstructed or modified files.

Success Criteria

The Relvio brand asset system is successful when:

Every team uses approved Relvio assets.
The approved logo is never reconstructed.
Brand identity remains consistent across supported surfaces.
Flutter production assets remain organized and controlled.
Brand source files remain separate from production UI assets.
Missing assets are reported instead of invented.
AI coding assistants can identify which assets are approved and how they may be used.
Documentation does not create conflicting asset rules.
