# MVP Scope

## Purpose

This document defines what will be included in the first public version (MVP) of Atlas.

The MVP should solve the core problems exceptionally well without trying to do everything.

The goal is to validate the product, onboard early customers, and collect real-world feedback.

---

# MVP Goals

The first release should enable an organization to:

- Manage people
- Organize events
- Record attendance
- Assign and track follow-ups
- View basic reports
- Manage team members

If Atlas can successfully do these six things, the MVP is considered successful.

# Out of Scope

The following will NOT be built in the MVP:

- Mobile Push Notifications
- AI Assistant
- Public API
- Workflow Builder
- Integrations
- Accounting
- Donation Management
- Volunteer Scheduling
- Payment Processing
- Marketplace
- Multi-language Support

These features belong to future releases.

---

# MVP Success Metrics

The MVP is successful if:

- Organizations can onboard without assistance.
- Attendance can be recorded reliably.
- Follow-ups are completed efficiently.
- Reports provide useful insights.
- Users return regularly.

---

# Launch Criteria

Atlas MVP is ready for beta when:

- All core modules are complete.
- Critical bugs are resolved.
- Documentation is complete.
- Performance is acceptable.
- Security review is complete.

---

# Guiding Principle

When deciding whether to add a feature to the MVP, ask:

> Does this help organizations manage people more effectively today?

If not, it goes into the backlog.
