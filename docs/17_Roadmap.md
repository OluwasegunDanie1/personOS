# Product Roadmap

## Purpose

This roadmap outlines the planned phases for building and growing Atlas.

It is a guide, not a fixed timeline. Priorities may change based on customer feedback.

---

# Phase 0 — Planning ✅

Goal: Build a strong foundation before writing code.

Deliverables:

- Product Vision
- Product Strategy
- PRD
- Information Architecture
- Design System
- Database Design
- API Specification
- Engineering Standards
- Testing Strategy
- Security Plan

Status: Completed

---

# Phase 1 — Design

Goal: Design the complete user experience.

Deliverables:

- Brand Identity
- Logo
- Color Palette
- Typography
- Wireframes
- User Flows
- High-Fidelity UI
- Design Components
- Clickable Prototype

Success Criteria:

- Every screen is designed.
- User flows are validated.
- Design system is complete.

---

# Phase 2 — Backend Foundation

Goal: Build the platform foundation.

Features:

- Authentication
- Organizations
- Roles & Permissions
- Database
- API
- File Storage
- Logging

Success Criteria:

- Secure authentication
- Multi-tenant support
- Stable API

---

# Phase 3 — MVP Development

Goal: Build the first usable version.

Modules:

- Dashboard
- People
- Events
- Attendance
- Follow-ups
- Reports
- Settings

Success Criteria:

A church should be able to manage its daily operations using Atlas.

---

# Phase 4 — Private Beta

Goal: Test Atlas with a small number of organizations.

Activities:

- Onboard beta users
- Collect feedback
- Fix bugs
- Improve onboarding
- Measure product usage

Success Criteria:

- Positive user feedback
- Stable performance
- High engagement

---

Success Criteria:

---

# Long-Term Vision

Atlas should become the platform organizations rely on every day to manage relationships, events, and engagement.

Instead of replacing just one tool, Atlas should become the central workspace for managing people.

---

# Roadmap Principles

When deciding what to build next, ask:

- Does this solve a real customer problem?
- Will customers use it regularly?
- Does it fit our vision?
- Can we maintain it long-term?
- Is now the right time to build it?

If the answer is "No", it moves to the backlog.
