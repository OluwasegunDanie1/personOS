# Iconography

## Purpose

This document defines the icon system used throughout Atlas.

Icons improve recognition, reduce cognitive load, and make the interface easier to scan.

Icons should support the content—not replace it.

---

# Design Philosophy

Atlas icons should feel:

- Modern
- Clean
- Minimal
- Consistent
- Professional

Icons should never become decorative elements.

Every icon must communicate meaning.

---

# Icon Library

Atlas will use a single icon library across the entire application.

Recommended Library

**HugeIcons**

Alternative

- Lucide
- Phosphor
- Material Symbols

Do not mix icon libraries.

---

# Icon Style

Use:

- Rounded
- Outline
- Minimal

Avoid:

- Filled icons mixed with outline icons
- Cartoon icons
- Decorative icons
- Skeuomorphic icons

---

# Stroke

Standard Stroke

```
2px
```

The stroke width should remain consistent across all icons.

---

# Corner Radius

Icons should use soft rounded corners.

Avoid overly sharp or aggressive shapes.

# Empty State Icons

Empty states should use:

- Simple illustrations
- Large icons
- Soft colors

Never use error icons for empty states.

---

# Icon Usage Rules

Do

- Use icons consistently.
- Pair icons with labels where appropriate.
- Maintain spacing.
- Use semantic colors.

Don't

- Rotate icons unnecessarily.
- Stretch icons.
- Mix styles.
- Use icons as decoration.

---

# Accessibility

Icons must include accessible labels.

Example

Instead of:

```
Icon Button
```

Use

```
Delete Person
```

Screen readers should understand every interactive icon.

---

# Icon Spacing

Icon → Text

```
8px
```

Icon → Icon

```
16px
```

Icon → Button Edge

```
12px
```

---

# Animation

Interactive icons may animate subtly.

Examples

- Chevron rotation
- Loading spinner
- Favorite toggle
- Expand/Collapse

Animations should:

- Be fast
- Be smooth
- Never distract

---

# Custom Icons

Atlas should minimize custom icons.

Create custom icons only when:

- No suitable icon exists.
- The icon represents a unique Atlas feature.
- It becomes part of the brand identity.

---

# Future Brand Icon Set

Future versions of Atlas may include a custom branded icon family for:

- Journey
- Follow-ups
- Organization
- Growth
- Community
- Relationship Mapping

These icons should follow the same visual language as the rest of the system.

---

# Success Criteria

The icon system is successful when:

- Icons are instantly recognizable.
- Every icon communicates a clear purpose.
- Designers and developers use the same icon set.
- Icons remain consistent across every platform.
- The interface feels clean, intuitive, and professional.
