# Feature Evaluation Checklist

Before moving a feature into development, ask:

- Does it solve a real customer problem?
- Has it been requested by multiple users?
- Does it align with the product vision?
- Can it be maintained long-term?
- Is it worth delaying other priorities?

If the answer is "No", keep it in the backlog.

---

# Backlog Maintenance

Review this backlog:

- After every major release
- After customer interviews
- After usability testing
- During roadmap planning

Remove outdated ideas and re-prioritize based on customer needs.
